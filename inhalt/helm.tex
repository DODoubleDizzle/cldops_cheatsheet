\section{Helm}
Package manager for K8 (apt/brew-like).
\textbf{Chart:} files describing related K8 resources.
\textbf{Release:} running instance.
\textbf{Repo:} stores versioned charts.
\textbf{Architecture:} client-side only (Helm 3, no Tiller); talks to K8 API directly; release state stored in-cluster as \textbf{Secrets} (in the release's namespace).
\textcolor{ForestGreen}{\textbf{Pros:}} huge chart library, versioning/deps, lifecycle (install/upgrade/rollback), powerful Go-template logic.
\textcolor{red}{\textbf{Cons:}} templates hard to debug, ``spaghetti YAML'', opaque state, steep curve. Use for: distributing complex apps, consuming community charts, advanced logic. Avoid for: simple YAML tweaks.

\subsection{Commands}
% Placeholders are the exact arg names from helm.sh docs, one color per token.
\begin{lstlisting}[language=sh, escapeinside={@}{@}]
helm repo add|list|rm @\pName{NAME}@ URL    # repos
helm search repo @\pName{<r>}@ / @\pChart{<c>}@ -l                 # all versions
helm install -f vals.yaml @\pName{NAME}@ @\pChart{CHART}@ --version 1.2.3
helm upgrade @\pRelease{RELEASE}@ @\pChart{CHART}@
helm rollback @\pRelease{RELEASE}@ REVISION
helm uninstall @\pRelName{RELEASE\_NAME}@   # also drops history
helm list; helm history @\pRelName{RELEASE\_NAME}@
helm status @\pRelName{RELEASE\_NAME}@      # unknown|deployed|uninstalled|
  # superseded|failed|uninstalling|pending-{install,upgrade,rollback}
helm get all|values|notes|manifest @\pRelName{RELEASE\_NAME}@  # values=overrides
helm template @\pName{NAME}@ @\pChart{CHART}@ -f v.yaml --set k=v  # render local
helm install @\pName{NAME}@ @\pChart{CHART}@ --dry-run --debug     # simulate
helm lint CHART_PATH [--with-subcharts]
helm show chart|readme|values|all @\pChart{CHART}@
helm package CHART_PATH        # build .tgz, then helm push <pkg> <repo>
helm dependency update @\pChart{CHART}@    # pulls into charts/
\end{lstlisting}

\subsection{Chart Structure}
\texttt{helm create NAME} scaffolds: \\
\kf{Chart.yaml} metadata \\
\kf{values.yaml} default values \\
\kf{templates/} Go-template manifests \\
\kf{templates/\_helpers.tpl}  named templates (\texttt{\_}=not rendered) \\
\kf{templates/NOTES.txt}  rendered \& shown post-install \\
\kf{charts/} subcharts; \kf{crds/}  CRDs \\
\kf{.helmignore} ignore globs (\texttt{!}=negate)
\subsection{Chart.yaml fields:}
\kf{apiVersion} \texttt{v2}=Helm 3+, \texttt{v1}=legacy
\kf{name} chart name; \kf{version} chart version, SemVer2 (e.g.\ \texttt{1.4.0}) $\to$ \texttt{name-version.tgz}
\kf{type} \texttt{application} (default) \textbar{} \texttt{library} (helpers only, no install)
\kf{appVersion} app shipped (info-only)
\kf{kubeVersion} allowed range (e.g. \texttt{>=1.12 <1.14}); install fails if mismatch
\kf{dependencies} name/version/repository/condition/alias (\texttt{requirements.yaml} deprecated; lockfile on update)

\subsection{Templating}
\textbf{Objects:} \kf{.Release.\{Name, Namespace, Revision, IsInstall, IsUpgrade\}}, \kf{.Chart.*}, \kf{.Values}, \kf{.Files.Get}, \kf{.Capabilities}, \kf{.Template}. \\
\textbf{Values precedence} (low$\to$high, last wins): subchart \kf{values.yaml} $\to$ parent \kf{values.yaml} $\to$ \texttt{-f my.yml} $\to$ \texttt{--set k=v}; unset via \texttt{--set x=null}. \\
\textbf{Functions} (Sprig, \textasciitilde60+): \kf{upper}, \kf{quote}, \kf{default}, \kf{trunc}, \kf{toYaml}, \kf{indent}/\kf{nindent}. \texttt{|} pipes result as last arg. \kf{include} = pipeable \kf{template}. \kf{required} fails if empty. \kf{tpl} renders a string as a template. \\
\textbf{Flow:} \texttt{\{\{if\}\}\dots\{\{end\}\}} (false on \texttt{0/""/nil/empty}); \texttt{\{\{range \$k,\$v := .list\}\}}; \texttt{\{\{with .sub\}\}} rescopes \texttt{.} \\
\textbf{Whitespace:} \texttt{\{\{-} trims left, \texttt{-\}\}} trims right; \texttt{nindent N} = newline+indent. YAML indent is critical.
\begin{lstlisting}[language=yaml]
{{- define "app.name" -}}{{ default .Chart.Name .Values.name | trunc 63 }}{{- end }}
name: {{ include "app.name" . | quote }}
{{- required "set .Values.img!" .Values.img }}
resources: {{- toYaml .Values.resources | nindent 4 }}
\end{lstlisting}
